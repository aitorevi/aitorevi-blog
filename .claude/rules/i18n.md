# i18n conventions

Sistema bilingüe ES/EN. La fuente de verdad vive en `src/i18n/`:

- `src/i18n/ui.ts` — merge de todos los messages, exporta `const ui = { es: {...}, en: {...} }`
- `src/i18n/utils.ts` — exporta `t(lang, key)`, `getLangLocale(lang)`, `getLocalePath(path, lang)`, `getBlogSlugFromId(id)`, etc.
- `src/i18n/messages/` — un archivo por dominio: `blog.ts`, `home.ts`, `misc.ts`, `nav.ts`, `cv.ts`, `portfolio.ts`, `site.ts`, `katas.ts`
- `src/i18n/types.ts` — tipo `Lang = 'es' | 'en'`

## Lo que SIEMPRE va a i18n

- Cualquier string de UI visible al usuario
- Labels, botones, `aria-label`, tooltips, mensajes de error
- Uso en Astro: `t(lang, 'clave')`
- Uso en React: `import { t } from '@/i18n/utils'`

## `getLangLocale(lang)` en vez de ternario

Sustituye cualquier `lang === 'es' ? 'es-ES' : 'en-US'`:

```ts
import { getLangLocale } from '@/i18n/utils'

formatDate(date, getLangLocale(lang))
```

## URLs de páginas traducidas

- Para rutas cuyo slug difiere entre idiomas (ej: `/privacidad` vs `/en/privacy`), añadir clave `*.url` en el messages file correspondiente.
- **No** usar `getLocalePath` cuando el slug cambia — ese helper solo añade el prefijo `/en/`.

## Lo que NO va a i18n (se mantiene como ternario)

- Selección de campos YAML del content (`taglineEs`, `hrefEs`, `labelEs`, etc.) — son datos, no strings de UI.
- Selección de arrays de datos por idioma (testimonios, listas de contenido).
- Lógica de rutas (`alternateLang`, `hreflang`, `og:locale`).
- Flags booleanos `const isEs = lang === 'es'` usados para lógica de componente.

## Estructura de claves por dominio

| Prefijo de clave | Archivo |
|---|---|
| `blog.*` | `src/i18n/messages/blog.ts` |
| `home.*` | `src/i18n/messages/home.ts` |
| `misc.*`, `legal.*` | `src/i18n/messages/misc.ts` |
| `nav.*` | `src/i18n/messages/nav.ts` |
| `cv.*` | `src/i18n/messages/cv.ts` |
| `portfolio.*` | `src/i18n/messages/portfolio.ts` |
| `site.*` | `src/i18n/messages/site.ts` |
| `katas.*` | `src/i18n/messages/katas.ts` |

Para una sección nueva, crear `src/i18n/messages/<section>.ts` e importarlo en `ui.ts`.
